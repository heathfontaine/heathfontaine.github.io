Work-Related Projects

  • Chatbot

    • The motivation behind a Chatbot was surrounded by the question of "How might we leverage emerging technologies to assist an employee in saving time?" Therefore, the idea of a Chatbot that can answer question or complete tasks was formulated and is constantly being worked on. There are tasks employees do daily, and much time can be saved if the task can be done by a simple message to a Chatbot. Emloyees and departments reach out to our team in order to request functionality that can assist the company as a whole.

    The technologies leveraged for the Chatbot primarily include various services in AWS. The services include a new natural language processing tool Amazon Lex. Alongside Lex, we utilize Lambdas, CloudFormation, CloudWatch, Elasticsearch, and many more services.

    My primary role on the team was to create intents (or functions) for the Chatbot. Each intent requires a set of utterances or phrases related to the intent in order to train the bot to understand when the intent a user is requesting. After creating the list of utterances with the customer or intent requestor, I would program the functionality, do extensive testing to ensure that not only the next intent functions correctly, but also no previous intent broke when training the bot, and then demo to the customer.

  • RightTrack Tag

    • The RightTrack Program at Liberty Mutual is a 90 day, voluntary program for auto insurance customers where a user plugs in a device to their car that tracks various driving statistics. After the evaluation period, the customer would receive a discount on their auto insurance premium based on his or her driving statistics. The problem with what was in place is that some older cars are not compatible with OBD2 devices, and that could cause some poor customer experience. On top of that, it was not convenient for a user to check current driving stats. Therefore, with the new program of RightTrack Tag, there would be a new device that was compatible with all cars, and also an app where you can see your stats in real-time from a mobile device.

    We partnered with two external teams in order to take on this new program. One team was dedicated to the Tag device while the other was in charge of the new app. As the Liberty team, we were in charge of setting up a new program in our system for the pilot states and integrating our systems with the other two teams' systems in order to make sure every component of the project functioned together correctly.

    My primary role on the team was to create automated tests that would run after changes were made. This helped to reassure us that if new functionality broke after any deploy, we could catch the bug as early as possible. Many tests were made using Selenium. These tests helped to free up manual testing by the members of my team. On top of creating tests, I was also a point of contact with the external teams since I was usually the first to know if a bug was discovered by one of my tests.

  • Automated Data Gathering

    • One of my responsibilities at Forcepoint was to help automate the collection of data for a Cyber Assessment (similar to an environment health check). Using Python and Powershell, I made a tool that can run in a customer environment that finds all the relevant Forcepoint servers (anywhere from 4 - 120 seen so far), detemine the OS and collect relevant data (whether using Powershell scripts, API calls, basic linux commads, etc.) and do analysis. It sped up the steps to do each server manually by 75%.

  • Paramiko Shell Prototype

    • My job at Forcepoint was heavily focused on using PuTTY to SSH into appliances and do work there. The problem is it quite a process to log in as root (involving getting encrypted passwords and decrypting them multiple times). Using the Python paramiko library, I was able to make a prototype for a shell window that bypassed many of these steps and automated the password collecting and decrypting. The user could then navigate the appliance as root as if it were a regular PuTTY window.
